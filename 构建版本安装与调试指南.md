### 方法二：手动安装
1. 安装证书（一次性）：
   ```powershell
   # 以管理员身份运行 PowerShell
   $certPath = "D:\重构工程\CleanReader.Desktop-3.2302.1.0\src\CleanReader.Bundle\CleanReader.Bundle_TemporaryKey.pfx"
   $cert = New-Object System.Security.Cryptography.X509Certificates.X509Certificate2($certPath)
   $store = New-Object System.Security.Cryptography.X509Certificates.X509Store(
       [System.Security.Cryptography.X509Certificates.StoreName]::Root,
       [System.Security.Cryptography.X509Certificates.StoreLocation]::CurrentUser
   )
   $store.Open([System.Security.Cryptography.X509Certificates.OpenFlags]::ReadWrite)
   $store.Add($cert)
   $store.Close()
   ```

2. 安装 MSIX 包：
   ```powershell

   $msixPath = "D:\重构工程\CleanReader.Desktop-3.2302.1.0\src\CleanReader.App\bin\x64\Debug\net8.0-windows10.0.22621.0\Richasy.CleanReader_1.0.0.0_x64.msix"
   Add-AppxPackage -Path $msixPath -ForceApplicationShutdown
   ```

## 🐛 闪退问题诊断

如果应用安装后闪退，请按以下步骤排查：

### 1. 检查 Windows App Runtime
WinUI 3 应用需要 **Windows App SDK 运行时**。

**检查是否安装：**
```powershell
Get-AppxPackage | Where-Object { $_.Name -like "*WindowsAppRuntime*" }
```

**如果没有安装：**
- 下载地址：https://learn.microsoft.com/windows/apps/windows-app-sdk/downloads
- 安装 **Windows App SDK 1.8** 运行时

### 2. 检查事件查看器日志
应用闪退时，Windows 事件查看器会记录错误。

**查看方法：**
1. 按 `Win + R`，输入 `eventvwr.msc`
2. 导航到 **Windows 日志** → **应用程序**
3. 查找来源为 **Application Error** 的事件
4. 查看 **错误模块** 和 **异常代码**

### 3. 检查证书问题
确保临时证书已安装到 **受信任的根证书颁发机构**。

**验证证书：**
```powershell
Get-ChildItem Cert:\CurrentUser\Root | Where-Object { $_.Subject -like "*CleanReader*" }
```

### 4. 卸载冲突版本
如果之前安装过 CleanReader，可能存在冲突。

**卸载所有 CleanReader 包：**
```powershell
Get-AppxPackage | Where-Object { $_.Name -like "*CleanReader*" } | Remove-AppxPackage
```

**然后重新安装您的构建版本。**

### 5. 调试构建配置
您的构建配置可能有问题：

1. **检查目标框架**：应为 `net8.0-windows10.0.22621.0`
2. **检查平台**：应为 `x64`
3. **检查生成设置**：确保生成 MSIX 包时包含所有依赖

### 6. 代码问题诊断
如果您修改了书源解析器，可能存在代码错误：

**建议：**
1. 在 Visual Studio 中直接调试运行（F5）
2. 检查异常输出窗口
3. 查看 `NovelService` 项目中的书源解析相关代码

## 📋 常见错误与解决方案

### 错误 1：没有注册类 (Class not registered)
**原因**：缺少 Windows App Runtime 或运行时版本不匹配。
**解决**：安装正确的 Windows App SDK 运行时版本。

### 错误 2：证书不受信任
**原因**：临时证书未安装或未受信任。
**解决**：重新安装证书到受信任的根证书颁发机构。

### 错误 3：应用无法启动，立即退出
**原因**：可能是代码中的未处理异常。
**解决**：
1. 在 Visual Studio 中调试
2. 检查 `App.xaml.cs` 中的 `UnhandledException` 事件
3. 查看应用日志文件（如果启用）

### 错误 4：依赖项缺失
**原因**：MSIX 包未包含所有必要依赖。
**解决**：确保生成设置中勾选“生成应用捆绑包”。

## 🛠️ 高级调试

### 启用应用日志
在 `App.xaml.cs` 中添加日志记录：

```csharp
private void OnUnhandledException(object sender, Microsoft.UI.Xaml.UnhandledExceptionEventArgs e)
{
    // 记录异常到文件
    string logPath = Path.Combine(ApplicationData.Current.LocalFolder.Path, "crash.log");
    File.AppendAllText(logPath, $"{DateTime.Now}: {e.Exception}\n");
    
    e.Handled = true; // 防止应用崩溃
}
```

### 使用 Windows 调试工具
1. 下载 **Windows SDK**
2. 使用 **WinDbg** 或 **ProcMon** 监视应用行为

### 检查系统兼容性
确保 Windows 版本满足要求：
- **WinUI 3**：需要 Windows 10 版本 1809 或更高
